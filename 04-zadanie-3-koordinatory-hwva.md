# Задание 3.1–3.3 — настройка координаторов HW-VA, VPN, чат и почта

## Подготовка dst-файлов

Есть два варианта работы с dst-файлами для координаторов: **iso-файл** или **флешка**:

![Варианты dst-файлов](images/p30_064.jpeg)

![dst-файлы](images/p30_065.jpeg)

На флешку скидываем dst-файлы для координаторов.
> ⚠️ Не перепутать, какой dst-файл для какого координатора!

Подсоединяем флешку к **Net1-HW-VA**:

![Подключение флешки к HW-VA](images/p31_066.jpeg)

![Подключение носителя](images/p31_067.jpeg)

![Подключение CD/ISO](images/p31_068.jpeg)

> ⚠️ После настройки **не забыть приконнектить CD**.

## Инициализация первого координатора (Net1-HW-VA)

Логин **`user`**, пароль **`user`**.

> ⌨️ В Linux **пароли не отображаются звёздочками**. Вся работа с клавиатуры: стрелки, Enter, Пробел (выбор), Tab (перемещение).

![Меню инициализации HW-VA](images/p32_069.jpeg)

Нажимаем **2** → окно лицензии → **Yes**.
Далее **Next → Европа → Россия → часовой пояс (Москва)**:

![Часовой пояс](images/p32_070.jpeg)

Проверяем дату → **Next**. Выбираем **cd → далее**.

![Выбор источника dst (cd)](images/p33_071.jpeg)

Выбираем dst-файл для **координатора офиса**. Далее **Next**, набираем пароль.
Соглашаемся с инициализацией сетевого интерфейса. Выбираем (стрелки + Пробел) **статический IP**:

![Статический IP](images/p33_072.jpeg)

Настраиваем **два сетевых интерфейса**: один в сегменте 1, второй в сегменте 3:

![Два интерфейса](images/p33_073.jpeg)

> У первого координатора два интерфейса: сегмент 1 смотрит в свою 1-ю сеть; сегмент 3 соединяет два координатора в одну сеть.

**Первый интерфейс (сегмент 1):** `172.16.224.230` (маска `255.255.255.224`):

![IP первого интерфейса](images/p34_074.jpeg)

Активируем. Снова выбираем статический адрес.
**Второй интерфейс (сегмент 3, «Интернет»):** `10.8.248.10` (маска `255.255.255.0`) → **Активировать**:

![IP второго интерфейса](images/p34_075.jpeg)

> 3-й и 4-й адаптеры **НЕ** активируем.

Появится запрос на IP шлюза:

![Запрос шлюза](images/p35_076.jpeg)

Шлюзом указываем **IP второго координатора**:

![Шлюз = второй координатор](images/p35_077.jpeg)

Далее → далее. Переименовываем координатор:

![Переименование координатора](images/p35_078.jpeg)

Далее → далее (ничего не меняем) → **Yes** (соглашаемся на фаервол).

## Настройка соединения между координаторами

Выбираем вторую строку:

![Вторая строка](images/p36_079.jpeg)

Далее → далее. Выбираем интерфейс, настроенный на «Интернет» (для связи между координаторами):

![Выбор интерфейса](images/p36_080.jpeg)

В списке появляется наш второй координатор:

![Второй координатор в списке](images/p36_081.jpeg)

Для проверки выбираем **Админа** (у него всё настроено):

![Выбор Админа для проверки](images/p37_082.jpeg)

Координатор не знает его IP — вводим. По заданию это **Net1-AdminCA `172.16.224.231`**.
Запускаются тесты, при успехе:

![Успешная проверка](images/p37_083.jpeg)

Далее → далее → перезагрузка координатора. Входить необязательно — настроенный координатор работает сам.

При успехе связь появится в двух клиентах:

![Связь в клиентах](images/p38_084.jpeg)

## Второй координатор (Net2-HW-VA)

Выставляем сегменты **2 и 3**:

![Сегменты 2 и 3](images/p38_085.jpeg)

![Настройка второго координатора](images/p39_086.jpeg)

Инициализируем так же, как первый. dst-файл берём из iso.

- **Первый интерфейс (сегмент 2):** `10.10.20.130` (маска `255.255.255.128`)
- **Второй интерфейс:** `10.8.248.20`
- 3-й и 4-й интерфейс **НЕ** активируем:

![Интерфейсы второго координатора](images/p39_087.jpeg)

Шлюзом указываем **IP первого координатора** (тот, что смотрит в «Интернет»).
Next → next, название — **Координатор Филиала** (таблица 1).

## Проверка соединения

Выполняем проверку:

![Проверка соединения](images/p40_088.jpeg)

Выбираем проверку по второму интерфейсу (с цифрой 1). Выбираем **Координатор Офис** для проверки связи, указываем IP первого координатора, запускаем.

> Если не работает — выйти из фильтра проверки, выбрать отмену проверки, затем **Да** на вопросе о старте VPN-сервиса. ⚠️ Если нажать **Нет** — придётся инициализировать всё заново.

При успехе на **Net1-Admin** появится связь с **RemoteUser**:

![Связь с RemoteUser](images/p40_089.jpeg)

Должна появиться связь с Админом из первой сети, а также у AdminCA и OperCA — с координатором филиала и Пользователем 2 (в идеале и с пользователем 3 / астра).

### Диагностика

Координаторы должны пинговаться между собой:

![Пинг координаторов](images/p40_090.jpeg)

И VPN-соединение должно пинговаться:

![Пинг VPN](images/p41_091.jpeg)

> Если ничего не работает — имеет смысл откатить снапшот на 0 и сделать заново.

## Проверка связи между сетями (в отчёт)

Из первой сети во вторую и обратно выполняем **Проверку** — обязательно скриншоты в отчёт:

![Проверка сеть 1 → сеть 2](images/p41_092.jpeg)

![Проверка сеть 2 → сеть 1](images/p41_093.jpeg)

Обязательно скриншоты о наличии связи и видимости из одной сети в другую.

> **Сообщение** — это чат, **Письмо** — это деловая почта.

![Чат / почта](images/p42_094.jpeg)

Написанное и отправленное сообщение:

![Отправленное сообщение](images/p42_095.jpeg)

> Деловую почту можно открыть из меню Пуск, если не открывается сама при входящих.

Письмо в исходящих по зашифрованному каналу:

![Исходящее письмо](images/p42_096.jpeg)

В идеале — скриншоты получения почты и сообщений чата из **обеих** сетей (между пользователями Офиса и Филиала).
