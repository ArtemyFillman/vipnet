# Задание 2.2 / 2.3 — координаторы, клиенты, роли, связи и ключи

## Создание координатора офиса

Открываем **ЦУС → вкладка Координаторы → зелёная кнопка `+`**:

![Создание координатора](images/p18_037.jpeg)

Переименовываем координатора по заданию — в **CentralCoord** (таблица 1 задания):

![Переименование координатора](images/p19_038.jpeg)

Нажимаем **Создать**.

## Создание клиентов

Вкладка **Клиенты → `+`**. Пишем **NetAdmin** (см. таблицу 2 задания — связи координатора и клиентов):

![Создание клиента NetAdmin](images/p19_039.jpeg)

Нажимаем **Выбрать** → выбираем для клиента единственного пока **координатора офиса**.
По таблице 2 создаём ещё пользователя. В результате:

![Список клиентов](images/p20_040.jpeg)

## Координатор филиала

Вкладка **Координаторы** → создаём координатор для филиала (таблица 1):

![Создание координатора филиала](images/p20_041.jpeg)

Снова **Клиенты** → создаём клиента, выбираем для него **координатора филиала**:

![Клиент филиала](images/p20_042.jpeg)

## Роли узлов

Вкладка **Координаторы** → двойной клик по **координатору офиса** → **Роли узла → Добавить**:

![Роли координатора](images/p21_043.jpeg)

Второму координатору тоже добавляем роль. Выходим.

Открываем свойства клиента **CA-Operator → Роли узла → добавляем роли** (таблица 1):

![Роли клиента](images/p21_044.jpeg)

Проверяем роли у остальных клиентов по таблице 1. Выходим.

## Адреса во внешних сетях

Вкладка **Координаторы** → свойства **координатора филиала → Адреса во внешних сетях → Добавить**:

![Адреса во внешних сетях](images/p22_045.jpeg)

Это адрес из IP-плана (см. [Главная](index.md)).

## Межсерверные каналы

Свойства **координатора офиса → Межсерверные каналы → Добавить** координатор филиала
(нужно для межсерверного соединения):

![Межсерверный канал](images/p23_046.jpeg)

Сохраняем, выходим.

## Связи между пользователями

Вкладка **Пользователи** → открываем свойства каждого пользователя и настраиваем **Связи** по таблице 2:

![Связи пользователей](images/p23_047.jpeg)

## Создание справочников

**Справочники и ключи → Создать справочники**:

![Создать справочники](images/p24_048.jpeg)

Выбираем весь список. После — на вкладке **Клиенты** появится надпись **«Ожидаются ключи»**:

![Ожидаются ключи](images/p24_049.jpeg)

## Создание ключей в УКЦ

Открываем **УКЦ** (на Net1-AdminCA). Видим пользователей и сообщение о необходимости создать ключи.

Открываем свойства **NetAdmin** (двойной щелчок по имени узла) → **Пароль администратора → создать пароль**:

![Пароль администратора узла](images/p24_050.jpeg)

Создаём пароли для **каждого** сетевого узла.

Вкладка **Группы узлов** → выбираем **вся сеть → свойства → сменить тип пароля на Собственный** и задаём свой пароль:

![Тип пароля группы узлов](images/p25_051.jpeg)

Поднимаемся на **Сетевые узлы → выбрать все узлы → Выдать новый дистрибутив ключей**:

![Выдать дистрибутив ключей](images/p25_052.jpeg)

Задаём пароль пользователя для каждого узла (спросят 5 раз — для каждого узла), остальные поля оставляем пустыми по умолчанию.

Появится информация о созданных ключах:

![Созданные ключи](images/p26_053.jpeg)

## Раздача ключей по машинам

Копируем папку с ключами в **Общую папку** (или на флешку):

![Копирование ключей в общую папку](images/p26_054.jpeg)

Идём по виртуальным машинам и раскидываем нужные ключи:

![Раздача ключей](images/p26_055.jpeg)

Запускаем на Рабочем столе **ключ (.key/дистрибутив)**:

![Запуск ключа](images/p27_056.jpeg)

После установки ключа запускаем **ViPNet Client** (из Пуска):

![Запуск ViPNet Client](images/p27_057.jpeg)

Набираем пароль, входим. Повторяем на остальных двух ПК с ViPNet Client.

В одном сегменте два узла должны увидеть друг друга:

![Видимость узлов 1](images/p28_058.jpeg)

![Видимость узлов 2](images/p28_059.jpeg)

Соглашаемся на установку сертификатов, сети и т.п. (всплывающие окна).

> На этом этапе VPN-соединение есть **только внутри первой сети**.

## Фильтры открытой сети (для клиента ЦУС)

Чтобы корректно работал клиент ЦУСа, на клиенте настраиваем **фильтры Открытой сети**:

![Фильтры открытой сети](images/p29_060.jpeg)

Внизу справа — кнопка **СОЗДАТЬ**:

![Создание фильтра](images/p29_061.jpeg)

Выбираем **Пропускать трафик**, в качестве **Источника** добавляем **Мой узел**:

![Источник — Мой узел](images/p29_062.jpeg)

Сохраняем → появится **Фильтр 1**. Аналогично создаём **Фильтр 2** для источника **Другие узлы**. В списке — два фильтра:

![Два фильтра](images/p30_063.jpeg)

Нажимаем **Применить**.
